Proyecto SCADA / OT Industrial

Resumen completo por fases del laboratorio desarrollado en Packet Tracer

/* FASES 1 */

FASE 1 - Diseño inicial de la red SCADA

En esta fase se construyó la topología básica del entorno SCADA industrial.

  • Creación del router principal.
  • Creación del switch industrial.
  • Conexión de servidor SCADA.
  • Conexión de HMIs.
  • Conexión de PCs simulando PLCs.
  • Diseño de arquitectura OT básica.

Configuraciones Técnicas - FASE 1

Topología Física

  • Router Principal conectado al Switch por Fa0/1.
  • Servidor conectado a Fa0/5.
  • HMIs conectadas en Fa0/10 - Fa0/12.
  • PLCs conectados en Fa0/15 - Fa0/18.

Configuración Router Principal

enable
configure terminal

interface gig0/0
no shutdown

interface gig0/1
ip address 200.0.0.1 255.255.255.0
no shutdown

Configuración Servidor Principal

IP Address:      192.168.10.10
Subnet Mask:     255.255.255.0
Default Gateway: 192.168.10.1
DNS Server:      192.168.10.10
/* FASES 2 */

FASE 2 - Segmentación mediante VLANs

Se segmentó la red industrial usando VLANs para separar dispositivos críticos.

VLAN Red Uso
10 192.168.10.0 Servidores SCADA
20 192.168.20.0 HMI
30 192.168.30.0 PLC
40 192.168.40.0 PC Ingeniería

Distribución de puertos

Puerto Uso
Fa0/1 Trunk hacia Router
Fa0/5 - Fa0/9 Servidores SCADA
Fa0/10 - Fa0/14 HMI
Fa0/15 - Fa0/22 PLC

Configuraciones Técnicas - FASE 2

Creación de VLANs

enable
configure terminal

vlan 10
name SCADA

vlan 20
name HMI

vlan 30
name PLC

vlan 40
name INGENIERIA

Asignación de Puertos

interface fa0/5
switchport mode access
switchport access vlan 10

interface range fa0/10-12
switchport mode access
switchport access vlan 20

interface range fa0/15-18
switchport mode access
switchport access vlan 30

interface fa0/19
switchport mode access
switchport access vlan 40

Configuración Trunk

interface fa0/1
switchport mode trunk
/* FASES 3 */

FASE 3 - Router-on-a-Stick + ACL Industrial

Se configuró inter-VLAN Routing utilizando Router-on-a-Stick.

Subinterfaces configuradas

  • gig0/0.10 → 192.168.10.1
  • gig0/0.20 → 192.168.20.1
  • gig0/0.30 → 192.168.30.1
  • gig0/0.40 → 192.168.40.1

Luego se implementaron ACLs industriales.

Objetivo de la ACL

  • Permitir HMI → PLC.
  • Permitir HMI → SCADA.
  • Bloquear PLC → HMI.
  • Bloquear PLC → SCADA.

Esto simula políticas reales de ciberseguridad OT.

Configuraciones Técnicas - FASE 3

Router-on-a-Stick

interface gig0/0.10
encapsulation dot1Q 10
ip address 192.168.10.1 255.255.255.0

interface gig0/0.20
encapsulation dot1Q 20
ip address 192.168.20.1 255.255.255.0

interface gig0/0.30
encapsulation dot1Q 30
ip address 192.168.30.1 255.255.255.0

interface gig0/0.40
encapsulation dot1Q 40
ip address 192.168.40.1 255.255.255.0

ACL Industrial

ip access-list extended CONTROL_PLC

deny ip 192.168.30.0 0.0.0.255 192.168.10.0 0.0.0.255

deny ip 192.168.30.0 0.0.0.255 192.168.20.0 0.0.0.255

permit ip any any

interface gig0/0.30
ip access-group CONTROL_PLC in
/*Fase 4*/

FASE 4 - Separación IT / OT + DMZ Industrial

Se construyó una arquitectura basada en ISA-95 separando IT y OT.

Zona VLAN Red
OT 10,20,30,40 192.168.X.0
DMZ 50 192.168.50.0
IT 60 172.16.0.0

Elementos agregados

  • Servidor DMZ.
  • PC corporativa IT.
  • Firewall OT mediante ACL.
  • Bloqueo IT → PLC.
  • Bloqueo IT → SCADA.
  • Permitir IT → DMZ.

Configuraciones Técnicas - FASE 4

Creación VLAN DMZ e IT

vlan 50
name DMZ_INDUSTRIAL

vlan 60
name RED_IT

Asignación de Puertos

interface fa0/21
switchport mode access
switchport access vlan 50

interface fa0/22
switchport mode access
switchport access vlan 60

Subinterfaces Router

interface gig0/0.50
encapsulation dot1Q 50
ip address 192.168.50.1 255.255.255.0

interface gig0/0.60
encapsulation dot1Q 60
ip address 172.16.0.1 255.255.255.0

Firewall Industrial ACL

ip access-list extended FIREWALL_OT

deny ip 172.16.0.0 0.0.0.255 192.168.30.0 0.0.0.255

deny ip 172.16.0.0 0.0.0.255 192.168.10.0 0.0.0.255

permit ip 172.16.0.0 0.0.0.255 192.168.50.0 0.0.0.255

permit ip any any

interface gig0/1
ip access-group FIREWALL_OT in
/* FASES 5 */

FASE 5 - Redundancia de servidores SCADA

Se implementó redundancia básica de servidores industriales.

Servidor IP Rol
SCADA Principal 192.168.10.10 Servidor Principal
SCADA Backup 192.168.10.11 Servidor de Respaldo

Funciones implementadas

  • DNS redundante.
  • Dominio empresa.local.
  • Pruebas de conectividad.
  • Pruebas de caída del servidor principal.
  • Simulación de alta disponibilidad.

Configuraciones Técnicas - FASE 5

Servidor SCADA Backup

IP Address:      192.168.10.11
Subnet Mask:     255.255.255.0
Default Gateway: 192.168.10.1
DNS Server:      192.168.10.10

Configuración DNS Backup

Domain: empresa.local

Registro 1:
empresa.local -> 192.168.10.10

Registro 2:
empresa.local -> 192.168.10.11

Pruebas Realizadas

  • Ping entre servidor principal y backup.
  • Ping desde HMIs a servidores.
  • Prueba de caída del servidor principal.
  • Prueba de resolución DNS.
